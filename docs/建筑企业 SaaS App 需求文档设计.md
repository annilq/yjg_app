# 建筑企业 SaaS App 需求文档

## 1. 项目概述

本项目是一个为建筑企业设计的 SaaS 应用，旨在提供企业内部管理、工作流程处理、信息通知等功能。目前使用 Objective-C 开发，计划使用 Flutter 进行重构，以实现跨平台开发和更好的用户体验。

## 2. 核心功能模块

### 2.1 用户认证模块
- **登录功能**：支持账号密码登录，包含密码加密处理
- **令牌管理**：获取和管理用户登录令牌
- **密码重置**：通过短信验证码重置密码
- **密码修改**：支持用户修改密码

### 2.2 主界面模块
- **主菜单获取**：获取应用主菜单列表及其版本信息
- **工作流程列表**：获取用户待处理和已处理的工作流程
- **提醒信息**：获取系统提醒和通知
- **文档列表**：获取重要文档列表
- **数据列表配置**：获取数据列表的配置信息

### 2.3 通讯录模块
- **常用联系人**：获取用户常用联系人列表
- **完整通讯录**：获取企业完整通讯录，按部门组织
- **联系人详情**：获取单个联系人的详细信息
- **常用联系人设置**：设置或取消联系人的常用状态

## 3. 技术架构

### 3.1 前端技术
- **开发框架**：Flutter
- **状态管理**：Provider/BLoC
- **网络请求**：Dio
- **UI组件**：Cupertino

### 3.2 后端接口
- **API 基础 URL**：http://api.jianguanoa.com/1.0/
- **用户中心 URL**：http://uc.jianguanoa.com/
- **请求方式**：POST
- **认证方式**：基于时间戳和签名的认证机制

## 6. 功能流程图

### 6.1 登录流程
1. 用户输入账号、用户名和密码
2. 客户端加密密码
3. 发送登录请求
4. 服务器验证并返回用户信息
5. 客户端存储用户信息和令牌
6. 跳转到主界面

### 6.2 工作流程处理
1. 用户进入工作流程界面
2. 选择待办/已办/相关事项
3. 查看详情
4. 处理工作流程
5. 提交处理结果

### 6.3 通知管理流程
1. 用户进入通知界面
2. 查看未读通知
3. 点击通知查看详情
4. 标记为已读或全部已读

## 7. 性能优化

### 7.1 网络优化
- 使用缓存减少重复请求
- 实现请求队列和并发控制
- 网络状态监测和自动重连

### 7.2 界面优化
- 实现懒加载和分页加载
- 使用 Flutter 的性能优化特性
- 减少不必要的重建和重绘

### 7.3 安全优化
- 密码加密存储
- 网络请求参数加密
- 防止 XSS 和 CSRF 攻击

## 8. 兼容性和适配

- 支持 iOS 和 Android 平台
- 适配不同屏幕尺寸
- 支持离线操作和数据同步

## 9. 测试计划

- 单元测试：测试各个功能模块
- 集成测试：测试模块间的交互
- 性能测试：测试应用的响应速度和资源占用
- 兼容性测试：测试在不同设备和系统版本上的表现
