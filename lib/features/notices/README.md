# 通知模块 (notices)

## 模块功能

通知模块负责通知的展示和管理，包括系统通知、业务通知等功能。

## 目录结构

```
notices/
├── service/        # 数据服务
│   └── notices_service.dart     # 通知服务
├── models/          # 实体
│   └── notice_model.dart     # 通知模型
├── presentation/    # 表示层
│   ├── screens/     # 页面
│   │   └── notices_list_screen.dart      # 通知列表页面
│   └── widgets/     # 组件
└── providers/       # 状态管理
    └── notice_provider.dart    # 通知状态管理
```

## 依赖关系

- **核心模块**: 依赖 core 模块中的常量、主题、工具类和错误处理
- **共享模块**: 依赖 shared 模块中的通用组件和服务
- **认证模块**: 依赖 auth 模块中的认证状态

## 使用方法

### 通知列表

1. 用户通过底部导航栏或侧边栏进入通知模块
2. 通知列表页面展示系统通知和业务通知
3. 用户可以点击通知查看详细信息
4. 用户可以标记通知为已读

## 主要服务

### NoticesService

负责通知相关的服务，包括获取通知列表、标记通知为已读等功能。

## 状态管理

使用 Riverpod 进行状态管理，主要包括：

- `noticeProvider`: 管理通知列表状态

## 注意事项

- 通知模块需要及时更新通知状态，确保用户能够看到最新的通知
- 实现适当的错误处理和加载状态
- 考虑添加通知推送功能，及时提醒用户有新通知
